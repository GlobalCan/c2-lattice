C2 Lattice

Setup guide for the AI agent team coordination platform — v4.1

1 Install

One command registers the MCP server globally for all Claude Code sessions.

python install.py
What this does: Registers c2-lattice as a global MCP server in ~/.claude.json via claude mcp add --scope user. No pip dependencies needed.

To check installation status or uninstall:

python install.py --status    # check if installed
python install.py --uninstall  # remove cleanly

2 Start the Broker

The broker is the coordination hub. Start it in its own terminal — it runs until you close it.

cd /path/to/c2-lattice
python broker.py
Auto-start: The MCP server will auto-start the broker if it's not running. But starting it manually gives you the stderr log output.

3 Open Claude Code Sessions

Each terminal gets its own session with a unique auto-generated ID.

Architect (you)

$env:PEER_ROLE="architect"
claude

Workers (separate terminals)

claude

Workers auto-register as worker-xxxx with a random suffix. Each terminal window must be a separate window (not tabs in the same terminal) to guarantee separate MCP processes.

Important: Never create ~/.mcp.json for c2-lattice. It overrides the global config and causes all sessions to share the same peer ID.

4 Open the Command Center

http://127.0.0.1:7899/dashboard

The interactive dashboard where you control everything. Auto-refreshes every 10 seconds.

Pause / Resume Agents

Freeze any agent instantly. They'll stop on next tool call and wait for resume.

Send Messages

Direct messages to any peer from the browser. 7 categories including blocker + error.

Set Budgets

Token caps per agent with visual budget bars. Prevents quota burn from runaway agents.

Monitor Git State

See each agent's branch, dirty files, and commit. Conflict warnings when files overlap.

Ping Health Check

Instant health status. Stuck detection when heartbeat is >20s old.

Tabbed Views

Peers, Messages, Tasks, Runs, Memory, Conversations, Activity — all in one place.

5 Direct Your Team

Tell your architect session what to do. Example prompt:

You are the architect of a team. Use C2 Lattice MCP tools to:

1. Create a run: set_memory with key="current-run", value="feature-auth"
2. Create tasks with dependencies (blocked_by)
3. Monitor workers via list_peers, check_messages
4. When workers raise_blocker, respond with guidance
5. When workers request_review, approve or send feedback

Your peer ID is shown in view_dashboard. Workers will claim
tasks, complete them with artifacts, and check messages for
your feedback. Control everything from the browser dashboard
at http://127.0.0.1:7899/dashboard

8 Use with Agent Teams

C2 Lattice + Claude Agent Teams work together. Teams spawns agents, Peers monitors them.

# In your Claude Code settings (~/.claude/settings.json):
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
Complementary: Agent Teams handles spawning and task assignment. C2 Lattice adds the dashboard, audit trail, budget controls, git awareness, and escalation routing that Teams doesn't have.
