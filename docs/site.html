Git Awareness

Conflicts caught before they happen

Every agent reports its branch, dirty files, and last commit hash on each heartbeat. The broker cross-references all working trees in real time, so it knows the instant two agents touch the same file.

When overlap is detected, the broker fires a conflict warning to both agents before either one commits. The warning includes which files overlap and who holds what locks, giving agents enough context to self-coordinate or ask the architect for guidance.

Combined with file locks, this makes merge conflicts nearly impossible. Agents can work on the same codebase simultaneously without stepping on each other, because the system enforces awareness at the coordination layer.

Task DAGs

Dependencies that actually enforce order

Tasks aren't just a list. Each task can declare dependencies on other tasks, forming a directed acyclic graph. Blocked tasks are physically unclaimable until every predecessor completes.

When an agent finishes a task, all downstream tasks that depended on it automatically unblock and become available for claiming. This means you can fan out parallel work and converge it naturally, without manual orchestration.

Completion requires artifacts: a summary of what was done, files touched, tests run, and known risks. No agent can mark a task "done" with an empty result. This creates a built-in audit trail for every piece of work.

Budget Caps

Token spend with hard limits, not hope

Each agent gets a configurable token budget with real-time tracking. The broker enforces limits at the task-claim boundary: when an agent's budget is exhausted, it physically cannot claim new tasks. No more runaway loops burning through your API quota.

Visual progress rings in the dashboard show each agent's consumption at a glance. Green means healthy, amber means approaching the limit, and red means the agent has been auto-paused and needs attention.

Budget state persists across sessions and is included in the telemetry log, so you have a full accounting of token spend per agent, per run, per task. This makes cost attribution trivial for multi-agent projects.

Security

Defense in depth, not security theater

10 layers of protection. Audited by Claude + Codex. Designed against OWASP ASI07 (inter-agent communication).

HMAC Token Authentication

Every peer receives a signed HMAC-SHA256 token at registration. Every request is validated. No token, no access. Tokens embed peer identity and role — tamper-proof without the broker's secret.

Role-Based Access Control

Three permission tiers: system, architect, worker. Workers can't kill peers or access logs. Architects can control other agents. Identity enforcement prevents impersonation — sender_id must match the token.

Localhost Only

Binds to 127.0.0.1 exclusively. Zero network exposure. No ports open to external traffic.

Content Filtering

NFKC Unicode normalization + regex blocks: tool_use XML, function-call JSON, base64, long paths. Prevents prompt injection via messages.

Rate Limiting

10 messages per minute per peer. Sliding window. Prevents spam and abuse between sessions.

Denial Tracking

Tracks rejections per agent. Auto-pauses after 20 rejections. Named stop reasons for every state change.

PID Liveness

Dead peer detection via os.kill(pid, 0). Dead agents auto-cleaned, tasks reassigned, locks released.

Tool Risk Metadata

Every MCP tool annotated with readOnlyHint and risk level (low/medium/high). 8 read-only, 10 medium, 2 high.

Input Validation

ID regex validation, size caps (10KB messages, 100KB requests), numeric bounds clamping, path normalization.

Zero Dependencies

Python stdlib only. No pip install, no node_modules, no supply chain. The attack surface is the Python standard library.
